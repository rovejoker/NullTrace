免费代理池 — 可靠性保障策略

五层防护机制确保代理转发不掉链

分层防护流程

从代理获取到请求完成的全链路保障
▶ 第1层：入库前验证（Pre-flight Check）
新代理加入池子前必须通过可达性测试
→ 向 httpbin.org/ip 发请求，5秒超时，失败直接丢弃
▶ 第2层：周期性健康检查（Health Monitor）
后台协程每 60 秒对所有池内代理进行存活性探测
→ 连续 3 次失败的代理自动剔除，恢复后重新加入
▶ 第3层：请求级超时 + 自动重试（Request Retry）
单次代理请求超时（默认 10s），立即标记失败
→ 自动从池中选取下一个代理重试，最多 3 次
→ 对上游扫描工具透明，它只看到一次成功的连接
▶ 第4层：智能评分与加权（Quality Scoring）
每个代理维护分数（0-100），基于：响应延迟 + 成功率 + 连续失败次数
→ 分数高的代理被选中的概率更大
→ 新代理从 50 分起步，逐步验证或淘汰
▶ 第5层：池底保护（Floor Protection）
当可用代理数低于阈值（默认 5 个），触发紧急补充
→ 自动重新爬取 + 降低准入标准（超时放宽到 8s）
→ 池子完全枯竭时，降级为直连模式并告警

生命周期状态机

代理 IP 在池子中的状态流转
UNKNOWN
新发现
→
TESTING
验证中
→
ACTIVE
可用
⇄
UNSTABLE
连续失败
→
BANNED
已剔除

ACTIVE ← UNSTABLE：连续成功 2 次则恢复
UNSTABLE → BANNED：连续失败 5 次则剔除
BANNED → TESTING：冷却 30 分钟后可重新验证
