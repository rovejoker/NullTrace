Tor 网络集成 —— 补强免费代理隐匿性

Tor 作为独立的"匿名层"，可与任意代理源组合

核心思路：Tor 不是代理源，而是匿名转发层

三种 Tor 使用模式
模式 A：纯 Tor

所有流量 → 本地 Tor SOCKS5
→ Tor 网络 (3跳) → 目标

✓ 隐匿性最强
✓ 零成本
✗ 速度慢 (100-500ms)
✗ Tor 出口节点可能被目标封禁
模式 B：免费代理 → Tor

流量 → 免费代理(1跳)
→ Tor 网络 (3跳) → 目标

✓ 免费代理看不到你的目标
✓ 目标看到的是 Tor 出口 IP
✓ 双重隐匿
✗ 延迟叠加 (200ms + 300ms)
模式 C：Tor → 付费代理

流量 → Tor 网络 (3跳)
→ 付费住宅代理 → 目标

✓ 目标看到的是住宅 IP
✓ 住宅 IP 不易被封
✓ ISP 看不到你在用代理
✗ 成本最高、延迟最高

链路配置示例

用户可自由编排链路
# 纯 Tor 模式
proxy start --provider tor

# 免费代理 → Tor → 目标（推荐：免费代理隐匿增强）
proxy start --provider free --relay tor

# Tor → 付费代理 → 目标（推荐：出口用住宅IP不被封）
proxy start --provider paid --api-key xxx --entry tor

# 自定义链路：免费代理 → Tor → 自建中继 → 目标
proxy start --chain free,tor,self-relay

# 纯免费单跳（不需要 Tor）
proxy start --provider free --hops 1

系统要求

🧅 Tor 模式需要本地运行 Tor 服务

本工具不内置 Tor，而是连接到本地已安装的 Tor 守护进程：

  • Linux: apt install tor && systemctl start tor → 默认 127.0.0.1:9050
  • macOS: brew install tor && brew services start tor
  • Windows: Tor Expert Bundle 或 Tor Browser 自带的 tor.exe

启动时自动检测本地 Tor 端口是否可用，不可用时给出安装指引。
